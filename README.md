# TBRR (Text-Based Roguelite RPG)

「物語の分岐点で選択肢を選ぶ」ことを繰り返して進む、テキスト中心のRPGです。  
TRPGのようなダイス判定システムがあり、同じ選択でも結果が変わることがあります。  
短い1プレイを何度も楽しむ「ローグライト」形式を採用しています。

【この手のゲーム経験者向けの要約】<br>
古典的TRPGのようなシステムで、FTL: Faster Than LightやSlay the Spireのようなゲームの選択肢の部分だけを延々とやるゲームです。
Ruina 廃都の物語の影響も色濃く受けています。

## 技術スタック
- **言語**: Java 21
- **UIフレームワーク**: JavaFX
- **データ管理**: JSON (Gson)
- **ビルドツール**: Maven
- **配布形式**: jpackage (Native Installer/Image)
- **開発環境**: Eclipse 2024-12 (Pleiades), Google Antigravity

## 開発体制
このプロジェクトは**プログラミング学習を兼ねた個人開発**として、開発者が主導しAIをツールとして活用して制作されました。

### 開発者が担当
- ゲームの企画・設計・仕様決定
- **非電子ゲーム（TRPG）のルールセットをプログラムに落とし込む設計**
- コードの理解と意図した動作の実現
- シナリオのプロット・素案作成
- バグ修正と品質管理

### AIツールの活用
- **コーディング支援**: コード生成・リファクタリングの提案（採否は開発者が判断）
- **設計相談**: アーキテクチャやデザインパターンの検討
- **デバッグ支援**: エラー原因の調査・解決策の提案
- **文章作成**: シナリオの肉付け・校正
- **画像生成**: キャラクターイラスト等（プロンプト設計は開発者が担当）

> [!NOTE]
> AIはあくまで「ツール」として活用しており、最終的な設計判断とコードの責任は開発者にあります。

### 使用ツール
- **開発全般**: Claude (Anthropic), Gemini (Google)
- **ゲームデザイン補助**: ChatGPT (OpenAI), Grok
- **画像生成**: AI画像生成ツール（本リポジトリからは除外）

## ビルド方法（ソースコード確認用）
素材データがないため、このままビルドしてもゲームは正常に動作しませんが、コンパイルの確認は可能です。

```bash
mvn clean package
```
